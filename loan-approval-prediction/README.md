# 🏦 Loan Approval Prediction

A machine learning project that predicts whether a loan application will be **approved or rejected** based on applicant details such as income, credit history, education, and more.

---

## 📌 Problem Statement

Financial institutions receive thousands of loan applications daily. Manually reviewing each one is time-consuming and error-prone. This project automates the approval decision using classification ML models, helping banks make faster and more consistent decisions.

---

## 📊 Dataset

- **Modeled on:** [Kaggle - Loan Prediction Dataset](https://www.kaggle.com/datasets/altruistdelhite04/loan-prediction-problem-dataset)
- **Records:** 614 rows × 13 columns
- **Target Variable:** `Loan_Status` (Y = Approved, N = Rejected)
- **Note:** `data/loan_data.csv` is a synthetic dataset generated by `data/make_dataset.py`,
  built with the same columns/shape as the real Kaggle dataset and genuine
  underlying relationships (credit history, income, education, etc. actually
  drive approval) so the models below have real signal to learn from. **If you
  have access to the real Kaggle dataset, just drop it into `data/loan_data.csv`
  (same column names) and everything else still works as-is.**

### Features Used:
| Feature | Description |
|---|---|
| Gender | Male / Female |
| Married | Applicant married or not |
| Dependents | Number of dependents |
| Education | Graduate / Not Graduate |
| Self_Employed | Self employed or not |
| ApplicantIncome | Applicant's monthly income |
| CoapplicantIncome | Co-applicant's monthly income |
| LoanAmount | Loan amount requested (in thousands) |
| Loan_Amount_Term | Repayment term in months |
| Credit_History | Credit history meets guidelines (1/0) |
| Property_Area | Urban / Semi-Urban / Rural |


## 🔧 Fixes Applied

This project was missing pieces and had one real bug — here's what was fixed:

- **Broken folder structure:** `mkdir {data,src,notebooks}` had been run in a shell
  that doesn't expand braces, creating one literal folder named `{data,src,notebooks}`
  instead of three real ones. Replaced with proper `data/`, `src/`, `notebooks/` folders.
- **Missing dataset:** added `data/make_dataset.py` and generated `data/loan_data.csv`
  (see the Dataset note above).
- **Missing notebook:** added `notebooks/eda.ipynb` with a full exploratory analysis
  (distributions, missing values, approval rate by credit history/property area, etc.)
- **Silent bug in `src/model.py`:** `df['col'].fillna(value, inplace=True)` no longer
  mutates the real DataFrame under pandas' copy-on-write behavior (default since
  pandas 2.x) — it was silently a no-op, so missing values survived into training
  and crashed `LogisticRegression` with a `NaN` error. Fixed by reassigning
  (`df['col'] = df['col'].fillna(value)`).
- **Headless plotting:** added `matplotlib.use("Agg")` and swapped the blocking
  `plt.show()` for `plt.close(fig)` so the script runs the same in a terminal,
  CI, or notebook without hanging on a display window.

## 📈 Results

Measured after the fixes above, on the included synthetic dataset:

| Model | Accuracy |
|---|---|
| Logistic Regression | ~89% |
| Random Forest | ~87% |

### Key Findings:
- **Credit History** is the most important feature for loan approval
- Applicants with higher income and good credit are significantly more likely to get approved
- Graduate applicants have higher approval rates
